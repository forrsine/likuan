# 2026-07-05 开发记录

系统设计文档第 9 章“第 6～7 天 DFA”的第一天：完成 NFA 状态集合表示、子集构造、DFA 状态表和基础执行器。

## 今日完成

- 新增 `dfa` 模块，DFA 状态使用动态 NFA 状态位图表示。
- 实现 move + epsilon-closure 子集构造。
- 为每个 DFA 状态生成 256 字节确定转移表。
- 设置 4096 个 DFA 状态上限，防止状态爆炸。
- 实现无内存分配的 DFA 执行器，并保持最左、同起点最长语义。
- 新增 `RX_FLAG_DFA`，现有 match、search、findall 可切换 DFA 模式。
- 新增 `rx_dump_dfa` 和 DFA 状态转移表。
- 新增 `test_dfa`，验证状态数量、接受状态、最长匹配和状态表。

## 转换示例

```text
pattern               NFA states  DFA states
a|b                   6           3
a*                    4           2
([a-z]+)\d{2,4}       16          6
```

演示命令：

```powershell
.\build-manual\rx_dump_dfa.exe 'a|b'
.\build-manual\rx_dump_dfa.exe '([a-z]+)\d{2,4}'
.\build-manual\rx_cli.exe --dfa '([a-z]+)\d{2,4}' 'abc123'
```

## 当前限制

DFA 子集构造目前只闭包普通 epsilon 转移。包含 `^` 或 `$` 的模式会明确返回 `RX_EUNSUPPORTED`，不会静默产生错误匹配；NFA 模式仍完整支持锚点。该问题安排在 DFA 阶段第二天解决。
