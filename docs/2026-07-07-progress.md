## 算法流程

1. 将 DFA 补充一个仅用于计算的隐式失败状态。
2. 把所有状态划分为接受状态集合与非接受状态集合。
3. 从工作队列取出“分区 + 输入类”，计算能够转移到该分区的状态集合。
4. 如果某个现有分区只有一部分状态属于该集合，则把它拆成两个分区。
5. 将较小分区加入工作队列，继续细化，直到不能再拆分。
6. 每个最终分区生成一个新状态，并重建转移与起始状态映射。

## 状态数对比

```text
pattern                    subset DFA    minimized DFA
a|b                        3             2
a*                         2             1
^abc$                      6             5
([a-z]+)\d{2,4}           6             6
```

复杂量词模式保持 6 个状态，说明这些状态的后继语言确实不同，不能继续合并。

## 验证结果

使用本机 MSVC 工具链以 `/W4` 编译：

```text
lexer tests passed
parser tests passed
NFA tests passed
DFA tests passed
all tests passed
```

重点验证：

- `a|b` 的两个等价接受状态合并后，仍能匹配 `a` 和 `b`。
- `a*` 合并为单个接受状态后，仍保持同起点最长匹配。
- `^a$` 最小化后仍拒绝非零搜索起点和未到文本结尾的匹配。
- 重复调用最小化不会继续改变状态数量。
- API 中使用的是最小化 DFA，NFA/DFA 的 match/search 状态和区间一致。
